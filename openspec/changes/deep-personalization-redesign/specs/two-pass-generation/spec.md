## ADDED Requirements

### Requirement: Two-pass generation pipeline
The system SHALL implement a 2-pass LLM generation pipeline: Pass 1 generates a learning blueprint (persona summary + topic roadmap), Pass 2 generates detailed content in batches.

#### Scenario: Pass 1 blueprint generation
- **WHEN** user triggers path generation
- **THEN** the system makes a single LLM call that returns a persona summary and a roadmap of 20-25 topics with titles, descriptions, difficulty levels, and image keywords

#### Scenario: Pass 2 batched detail generation
- **WHEN** Pass 1 blueprint is complete
- **THEN** the system generates detailed content in batches of 5 topics per LLM call, with each call including the persona context from Pass 1

### Requirement: Strict content contract
Each generated topic SHALL contain exactly 5 chunks. Each chunk SHALL contain 1 base sentence (root sentence with blanks) and exactly 3 variable phrases (SubPhrases). No more, no fewer.

#### Scenario: Content contract verification
- **WHEN** a batch of topics is generated by the LLM
- **THEN** each topic has exactly 5 chunks, each chunk has exactly 3 variableChunks, and any output not matching the contract is retried

### Requirement: Generation produces 20-25 topics
Each personalized learning path SHALL contain between 20 and 25 topics, organized in a logical difficulty progression from easy to advanced concepts relevant to the learner's profile.

#### Scenario: Path with sufficient depth
- **WHEN** a new personalized path is generated
- **THEN** the resulting path contains at least 20 and at most 25 topics

### Requirement: Real-time generation progress
The system SHALL provide a progress endpoint that reports generation status (current step, total steps, current topic name) so the frontend can display a progress bar.

#### Scenario: Progress polling during generation
- **WHEN** the frontend polls `GET /api/onboarding/generation-progress/{pathId}` during generation
- **THEN** the response includes `{ phase, currentBatch, totalBatches, currentTopicName, isComplete }`

#### Scenario: Progress display in UI
- **WHEN** generation is running  
- **THEN** the frontend displays a progress bar with "Generating topic X/25..." text and updates every 2 seconds

### Requirement: Batch retry on failure
The system SHALL retry failed batch calls up to 3 times. Successfully generated batches SHALL be preserved even if subsequent batches fail.

#### Scenario: Partial generation recovery
- **WHEN** batch 3 of 5 fails after 3 retries but batches 1, 2 were successful
- **THEN** the learning path is saved with 10 topics from the successful batches, and the user is informed of partial completion
